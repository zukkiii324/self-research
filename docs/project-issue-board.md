# Project Issue Board

この文書は、プロジェクト全体の主要課題を `優先順位` と `重要度` 付きで管理するための一覧です。

## ステータス定義

- `Open`: 未対応
- `Planned`: 方針合意済み、未着手
- `In Progress`: 着手中
- `Blocked`: 外部条件待ち
- `Done`: 完了

## 優先順位の定義

- `P0`: 放置すると公開や継続運用が壊れる
- `P1`: 早めに直さないと品質や速度を大きく落とす
- `P2`: 中期で解消したい改善課題
- `P3`: 余力があるときに進める強化項目

## 重要度の定義

- `Critical`: 公開停止、誤配信、運用不能につながる
- `High`: 品質低下や読者体験の悪化が大きい
- `Medium`: 改善価値は高いが即時停止要因ではない
- `Low`: 将来の改善候補
