# n8nで「毎朝レポート自動生成」を作る最短ルート

業務自動化を始めたい人が最初に作るべきなのは、複雑なAIエージェントより `毎日同じ時間に動く小さな仕組み` です。

その代表が、毎朝のレポート自動生成です。

今日は、n8nで次の流れを作る最短ルートをまとめます。

- 毎朝9時に起動
- Google Sheetsから必要な数値を読む
- 要点を整形
- Slackに送る

この型を1本持っておくと、売上、問い合わせ件数、広告数値、記事更新数、採用応募数など、かなり多くの仕事に横展開できます。

## 先に全体像

最小構成はこれで十分です。

`Schedule Trigger -> Google Sheets -> Code or Edit Fields -> Slack`

n8n公式ドキュメントでも、Schedule Triggerは固定時刻実行のためのノードで、公開済みワークフローとして動かす前提です。ここを軸にすると、まず「毎日勝手に走る」状態を作れます。

## ステップ1: Schedule Triggerで時間を固定する

最初のノードは `Schedule Trigger` です。

n8n公式では、

- 日次、週次、月次の指定ができる
- Cronも使える
- ワークフローかインスタンスのタイムゾーン設定を使う
- 公開しないと定期実行されない

とされています。

ここで一番よくある事故は、`時間は合っているのにタイムゾーンが違う` ことです。日本運用なら、ワークフロー設定のタイムゾーンを最初に揃えたほうが安全です。

私なら最初はこう置きます。

- Trigger Interval: Days
- Trigger at Hour: 9
- Trigger at Minute: 0
- Workflow Timezone: Asia/Tokyo

## ステップ2: Google Sheetsから必要な行だけ取る

次に `Google Sheets` ノードで読み込みます。

ここでやるべきことは1つです。`必要な列だけに絞る`。

Google Sheetsをそのまま全部読みに行くと、あとで整形が面倒になります。たとえば記事運用レポートなら、最初はこの4列で十分です。

- 日付
- 記事本数
- PV
- 売上

日次レポートで重要なのは「完全なデータ」より「毎日同じ形式で届くこと」です。最初の1本は、完璧な分析よりもフォーマット固定を優先したほうが続きます。

## ステップ3: 整形はCodeノードより先に文面テンプレを決める

ここでいきなりAI要約を入れる人が多いのですが、最初は不要です。

まずは `Edit Fields` か `Code` で、次の形に整えます。

```text
本日のレポートです。
- 記事本数: 3本
- PV: 12,400
- 売上: 18,500円
- 前日差: +1,200円
```

ポイントは、`Slackで3秒で読める文面` にすることです。朝レポートは分析資料ではなく、状況確認の通知です。通知に長文は要りません。

## ステップ4: Slackは「見出し+箇条書き」で送る

Slack送信では、凝った装飾より次を守るほうが効きます。

- 1行目で何のレポートか分かる
- 重要指標は3〜5個まで
- 異常値だけ別行で出す

たとえばこうです。

```text
【日次コンテンツレポート / 2026-04-10】
- 公開本数: 1
- PV: 4,280
- 保存数: 52
- 売上: 2,300円
- 前日比: +18%
注意: 売上は伸びたが保存率は低下
```

これだけで十分使えます。

## AIを入れるのは4本目のノードではなく2本目の運用から

AIは便利ですが、最初から入れるとトラブルの切り分けが難しくなります。

順番はこうです。

1. Scheduleで毎日動く
2. Sheetsを読める
3. Slackに定型文を出せる
4. その後でAI要約を足す

つまり、`自動化の本体は定期実行とデータ整形` です。AIは最後の磨き込みです。

## つまずきやすいポイント

### 1. 公開していない

Schedule Triggerは、保存だけではなく `publish` が必要です。ここを忘れると「テストでは動くのに本番で動かない」が起きます。

### 2. タイムゾーンがずれている

n8n公式でも、Schedule Triggerはワークフロー設定かインスタンス設定のタイムゾーンを使います。Asia/Tokyoを明示しておくのが安全です。

### 3. Sheetsの列名が毎回変わる

スプレッドシート側の列名変更は事故の元です。自動化用シートは、人が見やすい表ではなく、機械が読みやすい固定列で作るのが基本です。

## 最初に作るべき3パターン

この型は次にそのまま増やせます。

- コンテンツ運用レポート
- 問い合わせ件数レポート
- 売上/案件進捗レポート

全部、`Schedule -> Data Source -> Formatting -> Notification` の型です。

## まとめ

n8nで最初に作るべき自動化は、派手なものではなく `毎朝レポート` です。

理由は単純で、毎日動くものは壊れたらすぐ気づけるし、価値も分かりやすいからです。自動化は、複雑さではなく再現性から始めたほうが失敗しません。

まずは1本、`毎朝9時にSlackへ定型レポートを送る` だけ作ってみてください。それだけで、業務自動化の見え方がかなり変わります。
