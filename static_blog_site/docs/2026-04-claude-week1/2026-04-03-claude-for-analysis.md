# Claudeを分析用途で使い切る3つの型

2026年4月3日、Claudeを分析・洞察アシスタントとして使いこなすための代表的な型を紹介します。Claude 3 は `数万人トークン` のコンテキストを扱えるため、毎日追加されるデータをまとめて投げても答えが崩れにくいのが強みです。

**型1：データ + 注記 + 質問**

1. CSV/スプレッドシートから600行以内の要約を作る（ Claude に生の CSV を渡すか、要約を先に作る ）
2. 観測したキー指標とその変化理由を箇条書きにして添える
3. `次週の注力を3項目挙げて` と質問する

**型2：仮説対比**

1. 仮説A / 仮説B を表形式でClaudeへ渡す（例：広告予算を増やす vs 新素材を試す）
2. その仮説に必要な前提・KPI・制約をコメント
3. 「これらの仮説の差分を整理し、実験設計を提案して」と頼む

**型3：ドキュメント比較 + 反論生成**

1. 既存の企画書や調査レポートを `excerpt` 形式で並べる
2. Claudeに「この文書が論理破綻しうるリスクを3つ挙げ、それぞれどう検証すべきか」を求める
3. 出力をチームレビュー議事録へ貼り、次回の議題にする

## メモ

- データ量が多いときは必ず `analysis_mode`（Claudeのreasoning mode）をONにし、Chain-of-Thoughtを明示的に求めることでモデルの振る舞いが安定します。
- 出力をそのまま信じず、分析結果は必ず `raw` テキストと `検証メモ` をセットで保存しておきます。
